An Event Driven JavaScript/TypeScript Framework.

Overview

  1. What is StructureTS
    StructureTS is a collection of classes written in TypeScript and also a workflow to create JavaScript applications.
  2. Who is StructureTS for?
    Me, Robert S. www.codebelt.com
  3. Why did you build it?
    To make my life easier.
  4. Can I use it?
    Sure.
  5. Is StructureTS based off another language?
    Yes, I am developing the api and conventions to match ActionScript because it is a solid ECMAScript object-oriented language.

StructureTS is built in TypeScript. If want to learn more about TypeScript checkout these links:

Core Classes

DOMElement.ts

One of the main classes you will be extending in StructureTS is the DOMElement class. The DOMElement will be used to create all our views when referring to DOM elements. It allows you to create, layout, add, remove and destroy children from views. Below is how you would create a view class in StructureTS.


///<reference path='com/codebelt/structurets/display/DOMElement.ts'/>

module myNamespace
{
    import DOMElement = StructureTS.DOMElement;

    export class ExampleView extends DOMElement
    {
        public CLASS_NAME:string = 'ExampleView';

        constructor()
        {
            super();
        }

        public createChildren():void
        {
            super.createChildren();

            // Create and add your child objects to this parent class.
        }

        public layoutChildren():void
        {
            // Layout or update the child objects in this parent class.
        }

        public enable():void
        {
            if (this.isEnabled === true) return;

            // Enable the child objects and add any event listeners.

            super.enable();
        }

        public disable():void
        {
            if (this.isEnabled === false) return;

            // Disable the child objects and remove any event listeners.

            super.disable();
        }

        public destroy():void
        {
            super.destroy();

            // Destroy the child objects and references to prevent memory leaks.
        }

    }
}

To make development faster I've created template the DOMElement class.

  • DOMElement Template for PhpStorm/Webstorm
  • DOMElement Template for Sublime Test

There is a specific lifecycle the DOMElement class follows:

createChildren() is where you create the child objects for the view.

This method gets called only once when the child view is added to another view. If the child view is removed and added to another view the createChildren method will not be called again.

By default your view class will be a div element:


public createChildren():void {
    super.createChildren();
}

But lets say you wanted the view to be a ul element your would do:


public createChildren():void {
    super.createChildren('ul');
}
Then you could nest other elements inside this base view/element. 

public createChildren():void {
    super.createChildren('ul', {id: 'myId', 'class': 'myClass anotherClass'});

    var li:DOMElement = new DOMElement('li', {text: 'Robert is cool'});
    this.addChild(li);
}
So that's cool but what if you wanted a block of html to be your view. Let's say you had the below inline Handlebar template in your html file. 

<script id="todoTemplate" type="text/template">
    <div id="htmlTemplatel" class="js-todo">
        <div id="input-wrapper">
            <input type="text" class="list-input" placeholder="{{ data.text }}">
            <input type="button" class="list-item-submit" value="Add">
        </div>
    </div>
</script>
You would just pass in the id of the template which in this case is "#todoTemplate". There is a second optional argument where you can pass data for the Handlebar template to use. 

public createChildren():void {
    super.createChildren('#todoTemplate', { data: this.viewData });

}
One more. Let's say you wanted the Handlebar templates in there own files and then pre-compiled them so they would be faster to use. With the StructureTS work flow you can do this. Then you just have to reference the path to the template. 

public createChildren():void {
    super.createChildren('templates/home/HtmlTemplate.hbs');

}
layoutChildren() is where you layout / update any of the child objects for the view.
This method gets called right after createChildren is called but it can be called manually later to update the children of a view. 

public layoutChildren():void {
    var contentHeight = this.getBounds('#js-content').height;
    var tabContentHeight = this.pageHeight - contentHeight;

    this.$element.find('.tabs-pane')
                 .height(tabContentHeight);
}
enable() is where you enable the child objects for the view.
This method is responsible for enabling event listeners and/or children of the containing view. This way if you have children views nested with other children view you can be sure to all children and sub children of a parent view with can be enabled. 

public enable():void {
    if (this.isEnabled === true) return;

    this.$element.addEventListener('click', this.onMediaClick, this);

    this._tabComponent.enable();

    super.enable();
}
disable() is where you disable the child objects for the view.
This method is responsible for disabling event listeners and/or children of the containing view. This way if you have children views nested with other children view you can be sure to all children and sub children of a parent view with can be disabled. 

public disable():void {
    if (this.isEnabled === false) return;

    this.$element.removeEventListener('click', this.onMediaClick, this);

    this._tabComponent.disable();

    super.disable();
}
destroy() is where you destroy the child objects for the view.
This method is responsible for cleaning up the view and getting the child objects ready for garbage collection. 

public destroy():void {
    super.destroy();

    this._tabComponent.destroy();
    this._tabComponent = null;
}

Stage.ts

The Stage class extends the DOMElement class. You will extend this class for your main class. The Stage class has one extra method for appending your application into the DOM. The method is call appendTo() and accepts a selector. The selector can be a element's id or class name or you can reference a tag name like "body". You can see an example here. 


<script>
    window.onload = function(event) {
        // Get class from your namespace.
        var Main = codeBelt.Main;

        var app = new Main();
        app.appendTo('.wrapper');
    }
</script>
